## 修复64台服务器POD划分
- **问题**：64台服务器被错误分成2个POD（每POD 32台）
- **修复**：hardware-engine.js POD划分逻辑调整：
  - `serverCount <= 64` → 1个POD（原来32台以下才1个POD）
  - `serverCount <= 128` → 2个POD（原逻辑不变）
  - 其余逻辑不变
- 修复后64台 B300 计算结果：podCount=1, serversPerPod=64, leaf=16, spine=8

## 侧边栏区域间距最终修复
- **问题**：算力↔存储间距越来越大，但存储↔组网间距仍无变化，各区域间隙分配不均
- **根因**：侧边栏容器 `<div class="... space-y-0 ...">` 的 Tailwind `space-y-0` 类优先级高于 `.config-group`，强制覆盖所有子元素的 `margin-top`/`margin-bottom`，导致 `.config-group { margin-bottom: 32px }` 在部分区域失效（特别是存储服务器与组网架构之间）
- **修复**：
  - `index.html`：移除侧边栏内容容器的 `space-y-0` Tailwind 类，不再由 Tailwind 强制控制子元素间距
  - `css/style.css`：`.config-group` 恢复 `margin-bottom: 24px`（去掉之前激进的 32px）
  - `css/style.css`：去掉 `.config-group-body` 临时添加的 `padding-bottom: 12px`
- 最终间距分配：
  - 算力服务器 ↔ 存储服务器：24px
  - 存储服务器 ↔ 组网架构：24px
  - 组网架构 ↔ 重新生成拓扑按钮：24px
  - 按钮 ↔ 硬件测算：约 36px（按钮 margin 16px + 硬件测算 margin-top 20px）

## POD压缩显示（数量>2时参照Leaf/Spine模式）
- **需求**：当POD数量多于2个时，参照Leaf/Spine压缩模式，只显示POD1、POD2和最后一个POD，中间用gap占位
- **修改文件**：
  - `js/layout-engine.js`：
    - 新增 `buildPodPositions(podCount)` 函数：≤2个POD全显，>2时返回 `[pod1, pod2, gap(省略数), lastPod]`，3个POD时无gap直接显示3个
    - `calcLayout()` 中用 `podPositions` 替代直接遍历所有POD
    - 画布宽度基于 `visiblePodCount`（最大4个位置）计算，不再随实际POD数无限扩展
    - gap POD 生成虚线背景矩形 + `← xN →` 标签 + "省略 N 个POD" 子标签
  - `js/svg-renderer.js`：`renderBgLayer()` 中检测 `podIndex === -1` 的gap POD，应用虚线边框和居中文字对齐
- **验证结果**：
  - 1个POD → 显示1个 | 2个POD → 显示2个 | 3个POD → 显示3个 | 4+个POD → 显示POD1+POD2+gap+lastPod
